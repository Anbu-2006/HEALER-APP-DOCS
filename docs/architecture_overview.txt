1. APPLICATION ENTRY FLOW
================================================================================

  User launches app
       │
       ▼
  MainActivity (page_1.xml)
  ├── Single "Start Accident Detection" button
  ├── On click: starts SecondActivity with FLAG_ACTIVITY_SINGLE_TOP

       │
       ▼
  SecondActivity (page_2.xml)
  ├── Shows capability check dialog (11 sensors scanned)
  ├── Requests permissions (RECORD_AUDIO required, LOCATION optional)
  ├── On grant: calls startDetection()
  │   ├── Creates TrainingLogger (CSV session file)
  │   └── Calls initializeDetectors() — wires up all 8 sensors
  └── Live dashboard: 18+ TextViews updating in real time


2. SENSOR LAYER (detection/ package)
================================================================================

  All sensors live under com.example.healer.detection.
  Each sensor wrapper follows the same pattern:
    - Constructor takes Context
    - hasSensor(): Boolean — checks hardware availability
    - start(...callbacks) — registers SensorEventListener
    - stop() — unregisters listener, nulls references
    - Provides confidence boost or multiplier for fusion scoring

3. DETECTION & FUSION LOGIC (in SecondActivity)
================================================================================

  Five event timestamps are tracked:
    - shakeDetectedTime     (ShakeDetector.onCrashDetected)
    - soundDetectedTime     (SoundDetector threshold exceeded)
    - gyroDetectedTime      (GyroDetector peak detected)
    - lightDarknessTime     (LightSensor sudden darkness)
    - rolloverTime          (GravitySensor rollover detected)

  Fusion algorithm — checkForAccident():
    1. Collect all 5 timestamps into a list
    2. hasPairWithinWindow(1500ms):
       - For every pair (i, j) where i < j:
         if both != 0 and |t_i - t_j| < 1500ms → correlated
    3. If correlated AND ActivityDetector says NOT walking:
       → ACCIDENT CONFIRMED → showAccidentAlert()
    4. If user was WALKING → suppress (likely false positive)

  Probability scoring — calculateProbability():
    ┌─────────────────────────────┬──────────┐
    │ Signal                      │ Points   │
    ├─────────────────────────────┼──────────┤
    │ Impact detected (any type)  │ +30      │
    │ G-force > 2.5g              │ +25      │
    │ Sound > -50 dBFS            │ +15      │
    │ Secondary motion confirmed  │ +5       │
    │ Proximity: NEAR (pocket)    │ +15      │
    │ Light: <20% baseline        │ +10      │
    │ Light: <40% baseline        │ +5       │
    │ Gravity: upside down        │ +20      │
    │ Gravity: orientation change │ +10      │
    ├─────────────────────────────┼──────────┤
    │ Activity Multiplier         │          │
    │   IN_VEHICLE                │ ×1.5     │
    │   STILL / UNKNOWN           │ ×1.0     │
    │   WALKING                   │ ×0.3     │
    └─────────────────────────────┴──────────┘
    Final = clamp(sum × multiplier, 0, 100)


4. DATA LAYER (TrainingLogger)
================================================================================

  CSV Schema:
    timestamp, uptime_ms, acc_x, acc_y, acc_z, g_force, dbfs,
    gyro_x, gyro_y, gyro_z, speed_mps, impact_type, event

  Event types:
    - "sensor"  → continuous accelerometer samples (every 50ms)
    - "sound"   → microphone dBFS readings (every 200ms)
    - "gyro"    → gyroscope rotation values
    - "crash"   → confirmed impact event with type

  Storage: Android/data/com.example.healer/files/logs/session_<timestamp>.csv
  Threading: single-thread executor, buffered writes, flush every 32KB

6. LIFECYCLE MANAGEMENT
================================================================================

  SecondActivity lifecycle:
    onCreate   → inflate layout, bind views, wait for user
    Start tap  → detect capabilities → show dialog → request permissions
                 → startDetection() → initializeDetectors()
    onPause    → stop all 8 sensors (save battery)
    onResume   → if detectionRunning: re-create logger + re-init detectors
    Stop tap   → stopDetection() → stop all sensors, close logger, reset state


7. PROPOSED ML INTEGRATION (PLANNED)
================================================================================

  New packages to add:
    com.example.healer.ml/
      ├── AccidentClassifier.kt    — TFLite model loader & inference
      ├── SignalProcessor.kt       — noise filtering, resampling, normalization
      └── FeatureExtractor.kt      — raw windows → feature vectors

    com.example.healer.service/
      └── DetectionService.kt      — foreground service for background detection

    app/src/main/assets/
      └── accident_model.tflite    — trained model file

  ML Pipeline:
    Raw sensors → SignalProcessor (clean, resample to 50Hz, normalize)
               → FeatureExtractor (2s windows, time/freq/cross features)
               → AccidentClassifier (TFLite inference)
               → DetectionCoordinator (ML + rules → final decision)

  Model specs:
    - Architecture: 1D-CNN or small GRU/LSTM
    - Input: [1, window_size, num_features]
    - Output: softmax [NORMAL, ACCIDENT, FALL, DROP]
    - Size: <2 MB (int8 quantized)
    - Latency: <50ms on mid-range device

  Training plan:
    1. Collect labeled data via TrainingLogger (existing)
    2. Train offline in Python (TensorFlow/PyTorch)
    3. Export to .tflite with int8 quantization
    4. Integrate on-device via AccidentClassifier
    5. Fuse ML output with existing rule-based scoring
